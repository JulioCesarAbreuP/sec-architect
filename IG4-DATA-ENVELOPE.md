# IG4-DATA-ENVELOPE.md

## IG4 Data Envelope & Evidence Contract

This document defines the data envelope and evidence contract for deterministic, auditable, and secure data handling in SEC_ARCHITECT, as required by IG4 standards.

### 1. Data Envelope Structure
- All evidence and trace artifacts are wrapped in a versioned JSON or Markdown envelope.
- Envelope includes:
  - `version`: Envelope schema version
  - `timestamp`: ISO 8601 UTC
  - `source`: Module or engine name
  - `input_hash`: SHA-256 of input data
  - `output_hash`: SHA-256 of output/result
  - `evidence_type`: e.g., validation, remediation, threat, test
  - `actor`: Automated (engine) or human (user/committer)
  - `trace_id`: Unique identifier for audit trail

### 2. Deterministic Evidence
- All evidence is generated by pure, deterministic functions (see validation-engine.js, iac-engine.js).
- Re-running with the same input always produces the same envelope and hashes.

### 3. Audit Trail
- All envelopes are versioned and stored in the repository for full traceability.
- CHANGELOG.md and ADRs reference evidence artifacts for each architectural or security event.

### 4. Example Envelope (JSON)
```json
{
  "version": "1.0",
  "timestamp": "2026-04-04T12:00:00Z",
  "source": "validation-engine.js",
  "input_hash": "...",
  "output_hash": "...",
  "evidence_type": "validation",
  "actor": "engine",
  "trace_id": "uuid-v4"
}
```
